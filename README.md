# RZD Seat Assistant — бесплатный Render Web Service

Веб-панель для проверки заданных поездок РЖД. Аккаунт, пассажир и две поездки
уже встроены в эту персональную сборку. Проект настроен для **Free Web Service
без persistent disk**.

## Что делает сервис

- проверяет включённые поездки с заданным интервалом;
- ищет поезд, вагон и точное место;
- при доступности выбирает место, заполняет пассажира и пытается дойти до оплаты;
- **не нажимает кнопку оплаты**;
- поддерживает ручную проверку, журнал и диагностические скриншоты;
- выполняет одну проверку сразу после каждого запуска или пробуждения.

## Ограничения бесплатного Render

Free Web Service засыпает после 15 минут без входящих запросов. При засыпании,
рестарте или новом deploy удаляются SQLite, cookies, загруженная сессия РЖД и
скриншоты. Поэтому:

- встроенные аккаунт, пассажир и поездки восстановятся автоматически;
- изменения, сделанные только через веб-панель, могут исчезнуть;
- CAPTCHA и авторизация могут потребоваться повторно;
- внутренний планировщик не работает, пока сервис спит;
- сам по себе бесплатный Render не гарантирует проверку строго каждые два часа.

Когда сервис получает HTTP-запрос и просыпается, он сразу запускает проверку.
После этого проверки идут каждые два часа, пока процесс остаётся активным.

## Безопасность

Сборка содержит персональные данные и исходный пароль непосредственно в коде.
Загружайте её только в **приватный репозиторий**. После развёртывания смените
пароль РЖД и добавьте новый пароль как секретную переменную `RZD_PASSWORD` в
Render. Не публикуйте `APP_SECRET`, `ADMIN_PASSWORD`, `RZD_PASSWORD` и
`storage_state.json`.

Скрипт не обходит CAPTCHA. РЖД может менять интерфейс или блокировать IP
дата-центра, поэтому автоматическая работа не гарантируется.

## Развёртывание

### 1. Загрузите проект в приватный GitHub-репозиторий

Распакуйте ZIP. Содержимое папки `rzd_render_web_service` можно поместить в
корень репозитория.

```bash
git init
git add .
git commit -m "Initial RZD dashboard"
git branch -M main
git remote add origin https://github.com/YOUR_NAME/rzd-seat-assistant.git
git push -u origin main
```

### 2. Создайте бесплатный Web Service

В Render выберите **New → Web Service**, подключите репозиторий и установите:

| Поле Render | Значение |
|---|---|
| Language | `Docker` |
| Branch | `main` |
| Root Directory | пусто, если Dockerfile находится в корне |
| Dockerfile Path | `./Dockerfile` |
| Instance Type | `Free` |

Поля **Build Command** и **Start Command** оставьте пустыми. Команда запуска и
порт уже настроены в Dockerfile. Persistent disk добавлять не нужно.

### 3. Добавьте Environment Variables

В разделе **Environment** добавьте:

| Key | Value |
|---|---|
| `ADMIN_PASSWORD` | придуманный пароль для входа в веб-панель |
| `APP_SECRET` | случайная секретная строка не менее 32 символов |
| `DATA_DIR` | `/tmp/rzd-data` |
| `CHECK_INTERVAL_HOURS` | `2` |
| `CHECK_ON_STARTUP` | `true` |
| `APP_TIMEZONE` | `Asia/Vladivostok` |
| `HEADLESS` | `true` |
| `AUTO_RESERVE` | `true` |
| `COOKIE_SECURE` | `true` |

Для `APP_SECRET` используйте кнопку **Generate**. `PORT` добавлять не нужно —
Render создаёт эту переменную автоматически.

После смены пароля РЖД добавьте:

| Key | Value |
|---|---|
| `RZD_PASSWORD` | новый пароль РЖД |

При необходимости встроенный логин можно заменить переменной `RZD_LOGIN`.
Environment Variables сохраняются после сна и перезапусков, в отличие от
данных во временной SQLite.

### 4. Проверьте запуск

Нажмите **Create Web Service**, дождитесь статуса **Live** и откройте адрес
`*.onrender.com`. Войдите с `ADMIN_PASSWORD`. Для health check используется
`/health`.

На главной уже будут две поездки. Сначала нажмите **Проверить** у одной из них
и посмотрите результат и скриншот.

## Авторизация РЖД

Нажмите **Войти автоматически**. Если появилась CAPTCHA, введите её через
форму панели. Если вход с Render не проходит, локально выполните:

```bash
pip install playwright
playwright install chromium
python export_session.py
```

Полученный `storage_state.json` можно загрузить через **Настройки → Импорт
локальной сессии РЖД**, но на бесплатном Render он исчезнет при следующем сне
или перезапуске.

## Локальный запуск

Скопируйте `.env.example` в `.env`, замените секреты и выполните:

```bash
docker build -t rzd-seat-assistant .
docker run --rm -p 10000:10000 --env-file .env rzd-seat-assistant
```

Панель откроется по адресу `http://localhost:10000`.

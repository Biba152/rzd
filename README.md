# RZD Telegram Bot — обычный Render Web Service

Закрытый Telegram-бот с автоматическим оформлением фиксированного заказа РЖД
раз в два часа. Это обычный Docker Web Service: Blueprint и `render.yaml` не
используются.

## Уже встроено

- логин и пароль РЖД;
- токен Telegram-бота;
- разрешённые Telegram ID: `1143838304`, `5317465`, `5461520961`, `958854457`;
- маршрут Владивосток → Хабаровск-1 на 25.07.2026;
- поезд 005 «Океан», вагон 05, место 35;
- обратный маршрут на 29.07.2026;
- поезд 006 «Океан» в 19:40, купе, вагон 07, место 35;
- единственный сохранённый пассажир;
- автоматический запуск раз в два часа;
- Telegram-команды и кнопки;
- endpoint `/health` для проверки и пробуждения сервиса.

Все четыре Telegram ID имеют полный доступ. Уведомления и скриншоты
рассылаются каждому из них. Каждый пользователь должен один раз открыть бота и
нажать `/start`, иначе Telegram не позволит боту первым отправить ему сообщение.

## Команды

- `/start` — меню;
- `/status` — состояние и время запусков;
- `/run` — запустить оформление вручную;
- `/pause` — приостановить автоматические запуски;
- `/resume` — возобновить их;
- `/screenshot` — последний экран РЖД;
- `/config` — фиксированные параметры заказа;
- `/ping` — проверить работу;
- `/help` — справка.

## Как создать именно Web Service на Render

1. Распакуйте архив и загрузите **всё содержимое в корень GitHub-репозитория**.
   Файл `Dockerfile` должен находиться в корне рядом с `requirements.txt`.
2. На Render нажмите **New → Web Service**.
3. Подключите нужный GitHub-репозиторий.
4. В поле Runtime/Language выберите **Docker**.
5. Branch: `main` или ваша рабочая ветка.
6. Root Directory оставьте пустым, если файлы лежат в корне репозитория.
7. Выберите тариф **Free**.
8. Environment Variables не добавляйте — необходимые значения уже встроены.
9. Health Check Path при наличии этого поля: `/health`.
10. Нажмите **Create Web Service**.

Для Docker-сервиса отдельные Build Command и Start Command не нужны. Render
соберёт `Dockerfile`, а запуск возьмёт из его `CMD`:

```text
uvicorn app.main:app --host 0.0.0.0 --port $PORT --workers 1
```

Оставляйте только один экземпляр сервиса. Несколько экземпляров одновременно
запустят несколько Telegram polling-процессов и могут задвоить оформление.

## Необязательное переопределение

Встроенные значения всё ещё можно заменить переменными Render без изменения
кода:

- `RZD_LOGIN`;
- `RZD_PASSWORD`;
- `TELEGRAM_BOT_TOKEN`;
- `TELEGRAM_ALLOWED_IDS` — ID через запятую, пробел или `;`;
- `CHECK_INTERVAL_HOURS`;
- `RUN_ON_STARTUP`;
- `HEADLESS`;
- `APP_TIMEZONE`;
- `DATA_DIR`.

## Важно про бесплатный Render

Бесплатный Web Service может засыпать. Пока он спит, Telegram polling и
внутренний планировщик не работают. Запрос к `/health` пробуждает сервис; после
пробуждения приложение запускает пропущенную задачу, если с предыдущего запуска
прошло не меньше двух часов.

## Безопасность и ограничения

Секреты находятся в исходном коде ради минимальной настройки. Репозиторий должен
быть приватным. После проверки желательно перевыпустить Telegram-токен через
BotFather и заменить `DEFAULT_TELEGRAM_BOT_TOKEN` в `app/main.py`.

Бот нажимает «ОФОРМИТЬ ЗАКАЗ» и останавливается до оплаты. Если РЖД покажет
CAPTCHA, SMS-код или другую ручную проверку, бот не пытается её обходить — он
отправляет ошибку и скриншот.

# RZD Seat Assistant — веб-панель для Render

Веб-сервис с защищённой панелью, сохранённой сессией РЖД, поездками,
планировщиком раз в 2 часа, журналом, скриншотами и уведомлениями Telegram.

## Что он делает

- хранит настройки зашифрованно;
- автоматически проверяет включённые поездки каждые 2 часа;
- ищет указанный поезд, вагон и точное место;
- при доступности выбирает место, заполняет пассажира и пытается довести заказ до оплаты;
- **не нажимает кнопку оплаты**;
- после подтверждённого выхода на оплату ставит поездку на паузу;
- позволяет проверить поездку вручную из панели;
- сохраняет диагностический скриншот при проблемах.

## Важное ограничение

У РЖД нет публичного стабильного API автоматического оформления билетов.
Сайт может менять разметку, требовать CAPTCHA или блокировать IP дата-центра.
Поэтому код не может гарантировать вечную работу без настройки локаторов.

Скрипт не обходит CAPTCHA. Если автоматический вход не работает, используйте
`export_session.py`, войдите локально и загрузите `storage_state.json` в панель.

## Перед запуском

Вы уже публиковали пароль и паспортные данные. Сначала смените пароль РЖД.
Не коммитьте `storage_state.json` и не отправляйте его другим людям:
этот файл даёт доступ к авторизованной сессии.

## Развёртывание на Render

### 1. Создайте репозиторий

Распакуйте проект, затем:

```bash
git init
git add .
git commit -m "Initial RZD dashboard"
git branch -M main
git remote add origin https://github.com/YOUR_NAME/rzd-seat-assistant.git
git push -u origin main
```

### 2. Создайте Blueprint

В Render выберите:

1. **New → Blueprint**
2. подключите репозиторий;
3. подтвердите `render.yaml`;
4. задайте секрет `ADMIN_PASSWORD`.

`APP_SECRET` Render создаст автоматически.

Проект использует платный Web Service `starter` и persistent disk на 1 GB.
Это нужно, чтобы SQLite, cookies и скриншоты не пропадали при рестарте.

### 3. Откройте сайт

Перейдите на выданный адрес `*.onrender.com` и войдите с `ADMIN_PASSWORD`.

### 4. Заполните настройки

В разделе **Настройки** сохраните:

- новый логин и пароль РЖД;
- пассажира;
- Telegram Bot Token и Chat ID — необязательно.

### 5. Авторизуйте РЖД

На главной нажмите **Войти автоматически**.

Если появляется CAPTCHA:

- откройте показанный скриншот;
- введите код в форме;
- отправьте его.

Если вход с Render не проходит:

```bash
pip install playwright
playwright install chromium
python export_session.py
```

Затем загрузите получившийся `storage_state.json`:
**Настройки → Импорт локальной сессии РЖД**.

### 6. Добавьте поездки

Укажите дату, станции, поезд, вагон и место.

Лучше вручную выполнить поиск на официальном сайте РЖД и вставить готовую
ссылку страницы результатов в поле `search_url`. Это надёжнее заполнения формы.

### 7. Сначала протестируйте

Добавьте одну поездку и нажмите **Проверить**. Посмотрите статус и скриншот.
Только после успешного теста оставляйте автоматическую проверку включённой.

## Переменные Render

| Переменная | Назначение |
|---|---|
| `ADMIN_PASSWORD` | пароль входа в веб-панель |
| `APP_SECRET` | шифрование настроек и cookies панели |
| `DATA_DIR` | `/var/data` |
| `CHECK_INTERVAL_HOURS` | `2` |
| `APP_TIMEZONE` | `Asia/Vladivostok` |
| `HEADLESS` | `true` |
| `AUTO_RESERVE` | `true` |

Чтобы оставить только мониторинг без выбора места, задайте:

```env
AUTO_RESERVE=false
```

## Безопасность

- Панель защищена паролем и CSRF-токенами.
- Пароль РЖД и данные пассажира шифруются ключом, полученным из `APP_SECRET`.
- Скриншоты доступны только после входа в панель.
- Не публикуйте `APP_SECRET`, `ADMIN_PASSWORD`, storage state и persistent disk.
- Не запускайте несколько экземпляров сервиса: браузер и SQLite рассчитаны на один процесс.

## Почему не Render Cron Job

Cron Job не может подключить persistent disk. Тогда теряются SQLite и браузерная
сессия. Здесь планировщик APScheduler работает внутри единственного Web Service.
